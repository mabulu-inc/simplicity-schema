## Description

`getTriggers()` in `src/introspect/index.ts` uses `pg_get_expr(t.tgqual, t.tgrelid)` to read trigger WHEN clauses. This fails silently (returns NULL) for expressions that reference both `OLD` and `NEW`, such as `WHEN (OLD.* IS DISTINCT FROM NEW.*)`. This is a known PostgreSQL limitation — `pg_get_expr` cannot decompile expressions containing multiple relation variables.

The consequence is that on a second run, the planner sees `existing.when = undefined` vs `desired.when = "OLD.* IS DISTINCT FROM NEW.*"`, concludes the trigger has changed, and tries to recreate it every time. This breaks idempotency for any trigger with a WHEN clause referencing OLD/NEW.

## Current state

- `getTriggers()` (line ~577-621 of `src/introspect/index.ts`) queries `pg_get_expr(t.tgqual, t.tgrelid) AS when_clause`
- No error handling — NULL result silently becomes `undefined` on the TriggerDef
- `triggerNeedsRecreate()` in `src/planner/index.ts` (line ~1220) compares `(desired.when || '') !== (existing.when || '')` — always sees a diff when introspection returned NULL
- The E2E mixin test (`test/e2e/mixins.test.ts`) uses the timestamps mixin which has `WHEN (OLD.* IS DISTINCT FROM NEW.*)`, but may not test a second idempotent run against the same trigger

## Fix

Replace `pg_get_expr(t.tgqual, t.tgrelid)` with parsing the WHEN clause from `pg_get_triggerdef(t.oid)`. The `pg_get_triggerdef` function returns the full `CREATE TRIGGER ...` statement and correctly handles OLD/NEW references.

In `getTriggers()`, change the query to:

```sql
pg_get_triggerdef(t.oid) AS full_def
```

Then extract the WHEN clause from the full definition string. The format is:

```
CREATE TRIGGER ... WHEN (...) EXECUTE FUNCTION ...
```

Parse out the text between `WHEN (` and the matching `)` before `EXECUTE`. If no WHEN clause exists, the string won't contain ` WHEN (`.

## Produces

- Updated `src/introspect/index.ts` — `getTriggers()` uses `pg_get_triggerdef` to reliably read WHEN clauses
- New/updated tests in `src/introspect/__tests__/` or `test/e2e/` — test that a trigger with `WHEN (OLD.* IS DISTINCT FROM NEW.*)` is correctly introspected
- E2E idempotency test — run pipeline twice with a timestamps-style mixin trigger and verify second run produces zero operations

## AC

- `getTriggers()` correctly returns `when: "OLD.* IS DISTINCT FROM NEW.*"` for triggers using that clause
- A second pipeline run against the same schema with WHEN triggers produces zero operations (idempotent)
- All existing tests continue to pass
- `pnpm check` passes
