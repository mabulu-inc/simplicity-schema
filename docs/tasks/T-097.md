# T-097: View triggers (INSTEAD OF)

- **Status**: TODO
- **Milestone**: 29 — View Enhancements
- **Depends**: T-096
- **PRD Reference**: §4.4
- **Complexity**: standard

## Description

Add support for `INSTEAD OF` triggers on views, reusing the existing trigger infrastructure. PostgreSQL only allows `INSTEAD OF` timing on views (not `BEFORE`/`AFTER`), and these triggers enable updatable views backed by custom function logic.

## Changes

1. **`src/schema/types.ts`** — Add `triggers?: TriggerDef[]` to `ViewSchema`
2. **`src/schema/parser.ts`** — Parse `triggers` in `parseViewFile`, reusing existing `parseTriggerDef`
3. **`src/planner/index.ts`** — In `diffViews`, after planning the view itself, call the existing trigger diff engine (`diffTriggers` or equivalent) for each view that declares triggers. The view name acts as the table name. Introspect existing triggers on the view via `getTableTriggers`.
4. **`README.md`** — Document view triggers in the views section
5. **`docs/` updates** — Ensure any relevant docs reflect view trigger support
6. **Tests** — Parser round-trip; planner creates INSTEAD OF trigger on new view; planner detects trigger function change (drop + recreate); planner drops trigger when removed from YAML; idempotent when trigger already matches

## AC

- View YAML with `triggers` produces correct `CREATE TRIGGER ... INSTEAD OF ... ON view_name` operations
- Trigger diff (add, modify, remove) works for view triggers the same as table triggers
- Views without `triggers` field are unaffected (backward compatible)
- Trigger removal from YAML is treated as destructive (blocked without `--allow-destructive`)
- `pnpm check` passes
