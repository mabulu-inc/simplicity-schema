# T-099: Fix generate command not writing files to disk

- **Status**: DONE
- **Milestone**: 5 — CLI
- **Depends**: none
- **PRD Reference**: §3.3
- **Complexity**: light
- **Completed**: 2026-03-18
- **Completion Notes**: Fixed generate command to fall back to config.baseDir when --output is not set. Added integration tests verifying file writing behavior.

## Description

`schema-flow generate --db <url> --dir ./schema` reports "Generated N files" but writes 0 files to disk.

**Root cause:** `src/cli/index.ts:285` passes `parsed.output` to `generateFromDb()`, but `--dir` sets `parsed.dir`, not `parsed.output`. The `--output` flag is a separate arg. When `--output` isn't provided, `parsed.output` is `undefined`, and `src/scaffold/index.ts:84` skips the write block:

```typescript
if (outputDir) {  // undefined → skipped
  for (const file of files) { ... writeFileSync ... }
}
```

**Fix:** In the generate command handler (`src/cli/index.ts:285`), default to `parsed.dir` (which itself defaults to `./schema` resolved against cwd) when `parsed.output` is not set:

```typescript
parsed.output ?? parsed.dir,
```

This makes `--dir` the default output location for generate, which matches user expectation — generate writes into the same directory structure that `run`/`plan` reads from.

## Produces

- Fix in `src/cli/index.ts`
- Behavioral test: `generate` with `--dir` but no `--output` writes files to `--dir`
- Behavioral test: `generate` with explicit `--output` writes to `--output` (existing behavior preserved)
- Behavioral test: `generate` with neither flag writes to `./schema` (default)
