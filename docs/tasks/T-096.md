## Description

Add support for view options that generate a `WITH (...)` clause on `CREATE VIEW` statements. PostgreSQL supports options like `security_barrier`, `security_invoker`, and `check_option` on views.

## Changes

1. **`src/schema/types.ts`** — Add `options?: Record<string, string | boolean>` to `ViewSchema`
2. **`src/schema/parser.ts`** — Parse the `options` field in `parseViewFile`
3. **`src/planner/index.ts`** — Format `options` as a `WITH (key = value, ...)` clause in the `CREATE OR REPLACE VIEW` SQL generated by `diffViews`. Omit the clause when `options` is undefined or empty.
4. **`README.md`** — Document view options in the views section
5. **Tests** — Parser accepts/omits options; planner generates correct WITH clause; planner omits WITH when no options

## AC

- `options: { security_barrier: true }` produces `CREATE OR REPLACE VIEW ... WITH (security_barrier = true) AS ...`
- Omitting `options` produces no `WITH` clause (backward compatible)
- Multiple options are comma-separated
- `pnpm check` passes
